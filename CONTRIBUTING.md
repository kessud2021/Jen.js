# Contributing

Please read this guide before opening a pull request.

## Ways to Contribute

You can help by:
- Fixing bugs
- Improving documentation
- Adding examples
- Writing or improving tests
- Refactoring existing code
- Suggesting ideas or reporting issues

If you are new to the project, issues labeled `good first issue` or `documentation` are a good place to start.

## Getting Started

1. Fork the repository
2. Clone your fork
3. Install dependencies
4. Create a new branch for your change

Keep changes focused and scoped to a single purpose.

## Development Guidelines

- Follow the existing code style
- Prefer small, incremental changes
- Write clear and descriptive commit messages
- Avoid unrelated changes in the same pull request

If you are unsure about an approach, open an issue or discussion first.

## Pull Requests

When opening a pull request:
- Clearly describe what the change does
- Reference any related issues
- Explain why the change is needed
- Ensure the project builds and tests pass (if applicable)

Draft pull requests are welcome if you want early feedback.

## Documentation Contributions

Documentation improvements are highly appreciated.
This includes:
- Fixing typos or unclear wording
- Adding examples
- Clarifying APIs or behavior

Documentation-only pull requests do not require tests.

## Code of Conduct

Be respectful and constructive.
Harassment, personal attacks, or disrespectful behavior will not be tolerated.

## Questions

If something is unclear, feel free to open an issue or start a discussion.
